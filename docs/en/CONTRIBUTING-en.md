# CONTRIBUTING-en.md

Thank you for considering contributing to SimpleDAO! Whether it's reporting bugs, proposing new features, improving documentation, or submitting code, we welcome all forms of contribution.

## Reporting Issues
- Use the Issues feature on GitHub / Gitee.
- Clearly describe the problem, including:
  - SimpleDAO version used
  - Environment (JDK version, database type, etc.)
  - Steps to reproduce, expected vs. actual behavior
  - Relevant code snippets or logs

## Submitting Code
1. **Fork the repository**
2. **Create a branch**: from `main`, e.g., `feature/xxx` or `fix/xxx`
3. **Write code**: keep it simple and well-commented
4. **Test**: ensure existing tests pass, add new ones if needed
5. **Submit a Pull Request**: describe your changes and motivation

## Code Style
- Follow the existing style (simple, readable)
- Use 4-space indentation, avoid unnecessary complexity

## Documentation
- Documentation is in the `docs/` directory. Feel free to fix errors or add examples.
- Keep Chinese and English versions synchronized (if you change one, update the other).

## Questions
- Read existing docs and Issues first.
- If still unclear, open an Issue with the "question" label.

Thanks again for your support!
