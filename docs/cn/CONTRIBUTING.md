# CONTRIBUTING.md

感谢你考虑为 SimpleDAO 项目贡献！无论是报告 bug、提出新功能、改进文档还是提交代码，我们都欢迎。

## 报告问题
- 使用 GitHub / Gitee 的 Issues 功能提交。
- 请清晰描述问题，包括：
  - 使用的 SimpleDAO 版本
  - 运行环境（JDK 版本、数据库类型等）
  - 复现步骤、预期行为与实际行为
  - 相关代码片段或日志

## 提交代码
1. **Fork 仓库**
2. **创建分支**：从 `main` 创建你的特性分支，例如 `feature/xxx` 或 `fix/xxx`
3. **编写代码**：保持代码风格简洁、注释清晰
4. **测试**：确保现有测试通过，必要时添加新测试
5. **提交 Pull Request**：描述你的改动和动机

## 代码风格
- 遵循项目现有风格（简单、可读）
- 使用 4 空格缩进，避免不必要的复杂结构

## 文档改进
- 文档位于 `docs/` 目录，欢迎修正错误或增加示例
- 请保持中英文双语同步（如果修改中文，也请更新英文，反之亦然）

## 提问
- 先阅读现有文档和 Issues
- 若无答案，可在 Issues 中提问，注明“question”标签

再次感谢你的支持！
